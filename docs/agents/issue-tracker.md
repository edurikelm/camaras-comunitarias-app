# Issue Tracker

## Source Of Truth

Usar GitHub Issues como tracker principal salvo que el equipo defina otra herramienta.

## Workflow

- Cada bug o feature debe tener descripcion clara, contexto, criterio de aceptacion y riesgo.
- Las tareas grandes deben dividirse con `to-issues` antes de implementar.
- Las solicitudes ambiguas deben pasar por `triage` o `to-prd`.

## Agent Notes

- No inventar labels o estados nuevos sin actualizar `docs/agents/triage-labels.md`.
- Enlazar ADRs cuando un issue derive en decision arquitectonica durable.
